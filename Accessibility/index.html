Accessibility Testing on Lee County Libraries (LCL)

Name: Jatana Smiley
Class Project: Accessibility Testing For Lee County Libraries
Website: https://library.leecountync.gov/home
Date: January 28, 2024

The purpose of this report is to help identify possible website digital accessibility issues in order to prevent any access issues to this website via assistive technologies. The following accessibility checklist is based on the W3C WAI Easy Checks – A First Review of Web Accessibility I hope that you will find this report helpful and informative and find the recommendations to be actionable.

Accessibility Checklist

Website Page Titles

In the web page markup, the <title> is within the <head> tag. Good page titles are particularly important for orientation, they help people know where they are and move between pages open in their browser tab. The first thing screen readers say when the user goes to a different web page is the page title.

Results: 10 pages on the LCL website were evaluated for clear, unique HTML page titles. Each page has descriptive HTML title values. However, we do suggest that the title values have to be formatted as "Page Title at LCL", for example, "Summer Reading Program at LCL" so that it's obvious to users using the assistive technologies what website page they are on.

Image Text Alternatives

Image text alternatives ("alt text") (pictures, illustrations, charts, etc.) Text alternatives convey the purpose of an image, including pictures, illustrations, charts, etc. The alt attribute provides an alternative text for an image, if the user for some reason cannot view it. The reason can be a slow connection, an error with the image file, or if the user uses a screen reader.

Results: Every photo on the homepage except for 1 is missing the alt text. The 1 image that has text is simply labelled as "schedule." None of the images on the Library Staff and Board Member page have alt text. This means that people using assistive technologies would not know what the photo or graphic is communicating. We recommend that all photos and graphics have descriptive alt text.

Text:

Headings, Contrast Ratio ("color contrast") and Resizing Text should not only support visual aesthetics but also support the navigation and visibility of text content.

Results - Headings We tested the Homepage, Summer Reading page, and the Book Clubs page. None of these pages have an H1 header tag. This means that screen readers would not have a way to orient the top of the page. Additionally, there are no other header tags used to indicate portions of the page. There are 3 H4 tags being used at the bottom of the page in a stylized manner.

Results - Contrast Ratio We tested the Homepage, Summer Reading page and Book Clubs page. The WAVE color contrast tester flagged several portions of the pages as color contrast issues due to low contrast. The "More Search Tools" are a blue font on a white background. The footer consists of a black background with blue font. The low contrast can make it difficult for someone with a visual impairment to navigate the page. Suggestion is to change the color of the font and backgrounds to improve the level of contrast. Other suggestions include adding bolder font and adding underlines to indicate links.

Results - Resizing Text The LCL website scales up text size using keyboard commands very well. Text was increased to 200% and readability remained comprehensible on a desktop and iPhone 13.

Interaction:

Keyboard access and visual focus

Many people cannot use a mouse and rely on a keyboard to interact with web pages. Keyboard focus should be clearly visible and should follow a logical order through the page elements. Visible keyboard focus could be a border or highlight.

Results: Using only keyboard keys to navigate the LCL website was very difficult. There was no Skip to Main feature that allows keyword users to quickly skip to the main content section. Furthermore, Keyboard Focus was not obvious to us when we were using our tab and arrow keys. Only a portion of the page was indicated when that section was tabbed over. We recommend that the web team explore to make the LCL website more keyboard-only friendly.

Forms, labels, and errors (including Search fields):

Users are able to keyboard navigate to the different forms fields. Each form control has a label element with a for attribute that matches the value of the id attribute in the related control. For example:
  • Each form control has a label element with a for attribute that matches the value of the id attribute in the related control. For example:
    <label for="firstname">First name: </label>
    <input type="text" name="firstname" id="firstname" />
  • Each id is unique within the web page.

Results We tested the History and Genealogy page because it has a form. Using only the keyboard, we were able to go to each form field and press submit. The History and Genealogy page has a web form that contains titles and labels with a related unique ID.

General:

Moving, Flashing, or Blinking Content: Moving, flashing, or blinking content includes carousels (example carousel), ads, videos, auto-updating stock tickers, scrolling news feeds, and more. Users need to be able to control moving content, especially some people with attention deficit disorder or visual processing disorders.

Results: The LCL has an image carousel on the homepage that does not allow the user to stop the moving of images. The image gallery is not keyboard friendly. None of the photos have meaningful alt text. We recommend that the current image carousel is replaced with an accessible image carousel. No blinking or flashing content was located on the LCL website.

Multimedia (video, audio) closed captions and transcripts

Results: The video located on the Summer Reading page was tested for closed captions. We determined that the closed captions followed the spoken content with correct grammar. These captions are vital for individuals with hearing impairments.

Basic Structure Check: It helps you understand how some people "see" the web page differently. For this basic structure check, you look at the web page without images, styles, and layout. Web pages are often designed with multiple columns, sections, colors, and other visual aspects that help organize information for people who see the page in its default display. However, some people do not see the page this way.

Results: For this test, we used WAVE to remove all CSS styles in order to see the LCL website without any visual styling. We tested the Homepage and noticed that this page is heavy on graphics without alt text. Individuals using a screen reader would not be able to obtain any information from this page regarding your library or the services you provide.

Additional Notes

We did discover that the skip to main feature is reflected when all CSS styles were removed, but the feature was broken. Several of the webpages present a 404 error when we attempted to navigate to them. Those pages include the Computer Classes page and Learn a Language page.
